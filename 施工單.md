# Claude 流量  — 施工單

> 給 Claude Code 的規格。
> 基底:沿用已有的開源專案(server.js + index.html + 浮窗宿主腳本),只做「增量」,不要重寫。

## 專案定位
一個本機、免錢、輕量的常駐桌面工具。核心是「流量警報器」+「$100 credit 榨乾表」。
不爬瀏覽器、不做 Chrome extension、不外送任何 token。

## 已經現成、不要動的部分(沿用開源專案)
- `server.js` 讀 `~/.claude/.credentials.json` 的 OAuth token,打 `https://api.anthropic.com/api/oauth/usage`,
  拿 `five_hour` / `seven_day` / `seven_day_opus` 的 `utilization` + `resets_at`。
- token 只留後端、只綁 `127.0.0.1:3789`,絕不進前端。
- 已內建置頂浮窗模式 `?widget=1`(Windows `floating-widget.ps1` / macOS 對應版)。
- 已有 `scanClaudeContext()` 掃 `~/.claude/projects/*.jsonl` 拿每 session 的 input/cache token。

## V0.1 — 流量警報器(先出這個)
- 三條 bar:Session(5h)、Weekly(7d)、Opus 每週。資料來自現成 `oauth/usage`。
- 顏色門檻:≥70% 黃、≥85% 橘、≥92% 紅。
- **撞牆預警**:任一條 ≥90% 時,發**系統通知**(桌面 toast,不是只有變色),文案類似
  「⚠️ Weekly 91% — 建議開始收尾」。（專案目前只有變色,通知要新增。）
- 置頂浮窗沿用現成 `?widget=1`。
- 自動輪詢沿用現成快取/退避邏輯(別改短,避免 429)。

## V0.2 — Credit 榨乾表(路 A:本機精算,一定能做)
- 掃 `~/.claude/projects/*.jsonl`,**篩出 model 是 Fable 的訊息**,加總 input/output token。
- 換算已燒 credit:`input/1e6 * 10 + output/1e6 * 50`(**務必用 $10 / $50,別用舊的 $3/$15**)。
- 顯示:
  - 已燒 $X / $100、剩餘 $Y
  - 近 7 日平均每日燒錢速度 $Z/day
  - 預估耗盡天數 = 剩餘 / 每日速度
  - **預估耗盡日 vs 訂閱到期日** 比較 → 燒得完 😌 / 燒不完 🚨
- 明確標註:**此為「Claude Code 端」精算,不含 claude.ai 網頁上的 Fable 用量**(那部分本機沒 log)。
- 「金錢焦慮開關」:整塊可摺疊隱藏,預設收起。

## 硬規則
- 免錢:不得引入任何付費服務或執行期依賴。
- 輕量:維持零執行期依賴(Node 內建模組)。playwright 只是 devDependency,常駐不需要。
- 安全:token / credit endpoint 憑證一律只留後端,綁 127.0.0.1。
- 端點是非公開 API,可能改版:所有解析都要 graceful 失敗、沿用舊資料,不要整個崩掉。
